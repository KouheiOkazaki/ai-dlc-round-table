# INT-001 Requirements

## 機能要件

### FR-001: 備品マスタ管理
- 備品の新規登録（備品名、説明、返却期限日数）
- 備品情報の編集
- 備品の一覧表示（ID、名前、現在の貸出状況を含む）
- 備品の論理削除（貸出中の備品は削除不可）

### FR-002: QRコード発行・表示
- 備品登録時に一意のQRコードを自動生成する
- QRコードには備品IDを埋め込む
- 備品詳細画面でQRコードを表示・印刷できる
- QRコードの内容: アプリURL + 備品ID（例: `https://xxx.netlify.app/items/{itemId}/scan`）

### FR-003: QRコードスキャンによる貸出・返却
- スマートフォンのカメラでQRコードを読み取る（ブラウザ内カメラAPI使用）
- QRコードスキャン後、備品の現在状態を確認し、「貸出」または「返却」を選択できる
- 貸出時: 借用者名（任意テキスト入力）と返却予定日を記録する
- 返却時: 返却日時を自動記録する
- 貸出中の備品への再貸出は不可（エラー表示）
- 未貸出の備品への返却操作は不可（エラー表示）

### FR-004: 貸出状況ダッシュボード
- 全備品の現在の貸出状況を一覧表示する
  - 貸出中: 借用者名、貸出日時、返却予定日、経過日数
  - 返却済み（在庫あり）: 最終返却日時
- 返却期限超過の備品をハイライト（赤色等）で表示する
- 備品名での絞り込み検索

### FR-005: 貸出・返却履歴
- 全備品の貸出・返却履歴を新しい順に一覧表示する
- 各レコード: 備品名、借用者名、貸出日時、返却日時（未返却の場合は空欄）
- 備品IDによる絞り込み表示

## データモデル

### items（備品マスタ）
| カラム | 型 | 説明 |
|--------|-----|------|
| id | serial (PK) | 備品ID |
| name | text | 備品名（必須） |
| description | text | 備品説明（任意） |
| qr_code | text | QRコード用URL文字列 |
| default_loan_days | integer | デフォルト貸出日数（デフォルト: 7日） |
| created_at | timestamp | 登録日時 |
| deleted_at | timestamp | 削除日時（論理削除用、NULL=有効） |

### loans（貸出記録）
| カラム | 型 | 説明 |
|--------|-----|------|
| id | serial (PK) | 貸出ID |
| item_id | integer (FK) | 備品ID |
| borrower_name | text | 借用者名 |
| loaned_at | timestamp | 貸出日時 |
| due_at | timestamp | 返却予定日時 |
| returned_at | timestamp | 返却日時（NULL=未返却） |

## 入力・出力

### 入力
- 備品情報（フォーム入力）
- QRコードスキャン結果（備品ID）
- 借用者名（テキスト入力）
- 返却予定日（日付入力）

### 出力
- QRコード画像（SVG/PNG）
- 貸出状況ダッシュボード（HTML）
- 貸出・返却履歴一覧（HTML）
- 貸出・返却操作の成功/エラーメッセージ

## 制約・依存関係

### 制約
- QRコードスキャンはブラウザのカメラAPI（`getUserMedia`）を使用するため HTTPS 必須
- Netlify 環境にデプロイするため HTTPS は自動で担保される

### 依存関係
- 既存スキーマ (`comments` テーブル) はそのまま残す（他機能への影響なし）
- Drizzle ORM のマイグレーションで新テーブルを追加する
- QRコード生成ライブラリ（`qrcode` or `qrcode.react`）を追加する
- QRコードスキャンライブラリ（`@zxing/library` or `html5-qrcode`）を追加する
