# INT-001 Units Plan

## Unit 分割方針

機能的な凝集性と依存関係を考慮し、以下の3 Unitに分割する。
UNIT-001 がデータ基盤（DB スキーマ + 備品マスタ UI）を担い、後続 Unit がその成果物に依存する構造とする。

---

## Unit 一覧

### UNIT-001: 備品マスタ管理 + QRコード発行
**目的:** 備品の登録・編集・一覧表示、QRコード生成・表示機能を実装する。後続Unitが依存するDBスキーマもここで確立する。

**担当ユーザーストーリー:**
- US-001: 備品の登録
- US-002: 備品の編集
- US-003: 備品一覧の表示
- US-004: QRコードの表示・印刷

**主な成果物:**
- `items` テーブルスキーマ + Drizzle マイグレーション
- `/items` 一覧ページ（shadcn/ui）
- `/items/new` 登録ページ
- `/items/[id]/edit` 編集ページ
- `/items/[id]` 詳細ページ（QRコード表示）
- QRコード生成コンポーネント

---

### UNIT-002: QRコードスキャン + 貸出・返却操作
**目的:** スマートフォンのカメラでQRコードを読み取り、貸出・返却を記録する機能を実装する。

**担当ユーザーストーリー:**
- US-005: QRコードスキャンによる貸出
- US-006: QRコードスキャンによる返却

**依存:** UNIT-001（`items` テーブル、備品IDのURL構造）

**主な成果物:**
- `loans` テーブルスキーマ + Drizzle マイグレーション
- `/scan` スキャンページ（カメラ起動 + QR読み取り）
- `/items/[id]/scan` 貸出・返却操作ページ
- 貸出・返却 Server Actions

---

### UNIT-003: 貸出状況ダッシュボード + 履歴
**目的:** 貸出状況の可視化、返却期限超過アラート、履歴表示を実装する。

**担当ユーザーストーリー:**
- US-007: 貸出状況ダッシュボード
- US-008: 返却期限超過アラート
- US-009: 貸出・返却履歴

**依存:** UNIT-001（`items`）、UNIT-002（`loans`）

**主な成果物:**
- `/dashboard` 貸出状況ダッシュボードページ
- `/history` 貸出・返却履歴ページ
- 返却期限超過ハイライト表示ロジック

---

## Unit 間の依存関係

```
UNIT-001 (備品マスタ)
    |
    v
UNIT-002 (スキャン・貸出返却)
    |
    v
UNIT-003 (ダッシュボード・履歴)
```

直列依存のため並列開発は困難だが、各 Unit は独立してデプロイ・確認可能。
